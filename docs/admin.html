Event info

Special pricing (by phone)

Give a mobile number custom fixed prices per category. When that person registers online and enters this number, each category they pick shows the special price instead of the normal one (e.g. 5 KM = $15, 10 KM = $20). Enter 0 for a free entry — their registration lands as free on the dashboard, no payment step. Leave a category blank to charge the normal price. Numbers are matched after normalisation, so format doesn't matter.

Maintenance

Rebuilds the per-mobile index so a runner who signs in with their mobile + ref sees every registration filed under that phone. Run this once after enabling the sign-in feature, or any time the live data and the index drift apart.

Repairs registrations created before the ref-as-doc-id change shipped. Those legacy docs can't be looked up by ref code, so a runner clicking a sign-in / WhatsApp link sees "We couldn't find a registration with that reference code." This rebuilds each broken doc at the correct path (one per registration), updates the phone index, and removes the old doc. Safe to re-run anytime — it skips registrations that are already correct.

Scans every category for two runners sharing the same bib number. Bibs are meant to be unique within a category, so any match here is a clash to fix by editing one of the runners and giving them a free number. Read-only — it only reports, it never changes anything.

Re-evaluates every registration's sub-category against the current brackets on its category, then rewrites the saved label so old and new rows read the same way. Use this once after applying a new bracket preset — e.g. after switching to the IAAF birth-year labels. Idempotent (safe to re-run); only changes subQuestionAnswer on rows where it differs.

Backup & reset

One-click backup of the live data, and a guarded wipe to clear test data before opening real registration. Categories, theme, form fields, scanner, and admin/staff access are never touched by the wipe — only the per-runner data resets.

Download backup (JSON)

Saves all registrations (including soft-deleted), sponsors, categories, the phone index, and every config/* doc to a single JSON file you can store locally or commit privately.

Restore from a backup file

Loads a previously downloaded JSON and writes every doc back into Firestore (registrations, sponsors, categories, phone index, config). It runs in merge mode — same-ID docs are overwritten, new ones are added, and docs currently in Firestore that aren't in the file are left alone. For a clean rollback, run Wipe first, then Restore.

Wipe before going online

Removes every registration (live + soft-deleted) and the phone index, and resets each category's bibCursor so bib numbering starts from the range start again. Optionally also wipes sponsors. Run this once when you're done testing and ready to open real registration. It cannot be undone — take a backup first.
